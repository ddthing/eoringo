# 오늘 체크와 숙제 체크 모양 통일 설계

## 목표

오늘 탭의 `오늘 체크`와 숙제 탭에서 서로 다르게 보이는 체크 컨트롤을 작은 둥근 네모 형태로 통일한다. 체크 상태와 기존 동작은 그대로 유지한다.

## 현재 문제

두 화면은 같은 `TaskCheckControl` 컴포넌트를 사용하지만, 오늘 체크에서만 원형 크기 스타일을 덮어쓰고 있다. 이 때문에 동일한 숙제 입력 기능이 화면에 따라 서로 다른 컨트롤처럼 보인다.

## 변경 설계

- 오늘 체크의 개별 스타일 오버라이드를 제거한다.
- 두 화면 모두 `TaskCheckControl`의 기본 형태인 20px 크기의 작은 둥근 네모를 사용한다.
- 체크 아이콘, 완료 색상, 전환 애니메이션, 접근성 속성 및 상태 변경 로직은 수정하지 않는다.
- 시각적 크기만 통일하며 버튼의 실제 터치 영역은 기존 수준을 유지한다.
- 오늘 체크 외의 체크박스나 선택 컨트롤에는 영향을 주지 않는다.

## 타이포그래피 결정

현재 기본 본문색 `text-ink`는 순수 검정이 아니라 `rgb(31, 39, 48)`의 청회색 계열이다. 가독성과 시각적 위계가 적절하므로 이번 작업에서는 글자색을 변경하지 않는다. 강한 인상은 색상보다 제목에 사용하는 굵기에서 주로 발생하므로, 필요하면 별도의 타이포그래피 점검에서 보조 제목의 굵기만 선택적으로 조정한다.

## 회귀 방지

- 오늘 탭과 숙제 탭에서 체크 및 체크 해제가 동일하게 동작해야 한다.
- 완료 상태의 아이콘과 강조색이 유지되어야 한다.
- 390px 모바일 화면에서 컨트롤과 제목 정렬이 깨지지 않아야 한다.
- 테스트와 프로덕션 빌드를 실행한다.

## 변경하지 않는 범위

- 숙제 데이터 및 완료 처리 로직
- 체크 컨트롤의 애니메이션과 색상
- 전체 타이포그래피 토큰과 폰트 굵기
- 다른 입력 컨트롤의 모양
