# 기념일 UI 문구 통일 설계

## 목표

오늘 탭과 일정 탭의 기념일 영역에서 같은 제목, 동작 명칭, 빈 상태 문구를 사용해 사용자가 두 화면을 동일한 기능으로 인식하도록 한다.

## 화면 구성

두 화면의 기념일 영역 상단은 다음 문구로 통일한다.

- 분류 문구: `기념일`
- 제목: `기념일 관리`
- 추가 버튼: `추가` (아이콘과 함께 표시될 때 화면에는 `+ 추가`로 보임)

오늘 탭의 기존 `기록` 버튼은 `추가`로 변경한다. 일정 탭은 기존 추가 버튼의 의미와 동작을 유지하되 같은 문구를 사용한다.

## 빈 상태

등록된 기념일이 없을 때 두 화면 모두 달력 아이콘과 `등록된 기념일이 없습니다.` 문구 한 줄만 표시한다. 일정 탭 빈 상태에 있던 별도의 `기념일 추가` 버튼과 오늘 탭의 `다가오는 기념일이 없습니다.` 문구는 제거한다.

## 동작 및 데이터

기념일 추가, 수정, 삭제와 Home·일정 간 동기화 로직은 변경하지 않는다. 이번 작업은 표시 문구와 빈 상태 구성만 수정한다.

## 접근성

추가 버튼은 화면 문구와 접근성 이름이 모두 `추가` 의미를 갖도록 유지한다. 빈 상태 아이콘은 장식 요소로 처리해 문구가 중복 낭독되지 않게 한다.

## 검증

- 두 컴포넌트에서 헤더와 추가 버튼 문구가 동일한지 확인한다.
- 기념일이 없을 때 두 화면 모두 동일한 한 줄 문구를 표시하는지 확인한다.
- 전체 단위 테스트와 프로덕션 빌드를 실행해 기존 추가·수정·동기화 기능에 회귀가 없는지 확인한다.
