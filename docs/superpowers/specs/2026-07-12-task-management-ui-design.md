# 숙제 관리 UI 개편 설계

## 목표

숙제가 늘어나도 체크 화면의 단순함을 유지하면서 기본 숙제, 숨김 상태, 커스텀 숙제, 수주권을 한 곳에서 관리할 수 있게 한다. 사용자는 숙제 탭에서 체크에 집중하고 별도 관리 화면에서 검색, 필터, 활성화, 편집을 수행한다.

## 범위

- `/tasks/manage` 관리 전용 화면 추가
- 기존 숙제 탭의 하단 상세 관리 아코디언을 관리 화면 링크로 교체
- 기본 숙제 검색, 상태 필터, 초기화 종류 필터, 그룹 접기
- 숙제별 리셋 설명과 다음 초기화 정보 표시
- 수주권 현재 수량, 자동 누적 정보, 수동 편집 UI
- 기존 커스텀 숙제 추가·수정·삭제·활성화 기능 이동
- 390px, 430px, 768px 반응형 검증

숙제 완료 입력 화면, 리셋 데이터 구조, 알림 기능은 변경하지 않는다.

## 라우팅

- 새 경로: `/tasks/manage`
- 숙제 탭의 `상세 관리` 링크가 새 경로로 이동한다.
- 관리 화면의 뒤로가기는 브라우저 기록이 있으면 이전 화면으로 이동한다.
- 직접 URL로 진입한 경우에도 숙제 탭으로 돌아갈 수 있는 명시적 링크를 제공한다.
- 하단 내비게이션에서 숙제 항목은 `/tasks/manage`에서도 활성 상태로 표시한다.

## 화면 구조

### 헤더

- 뒤로가기 버튼
- 제목 `숙제 관리`
- 설명 `표시할 숙제와 초기화 주기를 관리합니다.`
- 현재 관리 대상 캐릭터를 표시하고 `CharacterSwitcher`로 전환한다.
- 캐릭터 이름은 선택 버튼에서 한 번만 표시한다.

### 상단 도구 영역

390px와 430px에서는 한 열, 768px에서는 수주권 카드와 검색·필터 패널을 두 열로 배치한다.

#### 수주권 카드

- 현재 수량을 `21 / 100` 형식으로 표시한다.
- `−1`, `+1` 버튼은 44px 이상의 터치 영역을 갖는다.
- 숫자를 누르면 숫자 입력 필드로 전환하며 0~100으로 제한한다.
- 포커스 이탈 또는 Enter에서 저장하고 Escape에서 취소한다.
- `다음 +3 · 오늘 21:00`처럼 다음 누적 시각을 표시한다.
- 자동 누적은 기존 `useAllowanceStore`를 사용한다.
- 100장일 때는 `최대 보유 중`을 표시한다.

#### 검색과 필터

- 숙제 이름, 설명, 메모를 검색한다.
- 상태 필터: `활성`, `숨김`, `전체`; 기본값은 `활성`이다.
- 초기화 필터: `전체`, `일일`, `주간`, `18시간`, `수동`; 기본값은 `전체`다.
- 검색어와 필터는 관리 화면 내부 상태로 유지한다. URL 쿼리에는 저장하지 않는다.
- 초기화 필터 의미:
  - 일일: `daily-midnight`, `daily-0500`, `daily-1700`
  - 주간: 화요일 17시와 토요일 21시 규칙
  - 18시간: `interval-18h`
  - 수동: `manual`
- 결과 건수를 `12개 숙제` 형태로 표시한다.

## 기본 숙제 관리

- 콘텐츠 그룹 순서는 기존 `taskGroupLabels`와 `groupOrder`를 유지한다.
- 각 그룹 헤더에 검색·필터 결과 기준 `활성 수 / 전체 수`를 표시한다.
- 기본 상태에서는 기존처럼 일부 그룹만 펼친다.
- 검색어가 있으면 결과가 있는 그룹을 자동으로 펼친다.
- 필터 결과가 없는 그룹은 렌더링하지 않는다.
- 전체 검색 결과가 없으면 필터 초기화 버튼이 포함된 빈 상태를 표시한다.

### 숙제 행

- 제목
- 리셋 설명:
  - 매일 00:00
  - 매일 05:00
  - 매일 17:00
  - 화요일 17:00
  - 토요일 21:00
  - 완료 후 18시간
  - 수동 초기화
- 고정 규칙은 `다음 초기화까지 3시간 20분`을 함께 표시한다.
- 완료 후 18시간 항목은 완료 시각이 있으면 남은 시간을 표시하고, 없으면 리셋 설명만 표시한다.
- 패션 체크는 `금요일 17:00부터 참여 가능` 보조 문구를 표시한다.
- 쿠로는 `최대 14일 보유` 보조 문구를 표시한다.
- 활성/숨김 스위치는 기존 캐릭터별 설정을 그대로 사용한다.
- 행 전체는 토글 버튼으로 사용하지 않는다. 스위치만 상태를 변경해 오작동을 줄인다.

## 커스텀 숙제

- 기본 숙제 아래 독립 섹션으로 둔다.
- 기존 추가, 수정, 삭제, 활성화 기능을 유지한다.
- 기존 데이터의 `daily`, `weekly`, `eighteenHours`, `manual` 선택 UI를 유지한다.
- 표시 문구는 새 리셋 라벨 유틸리티를 재사용한다.
- 관리 화면 필터는 커스텀 숙제에도 동일하게 적용한다.
- 삭제는 기존 확인 대화상자를 유지한다.

## 컴포넌트 구조

- `TaskManagementPage`: 라우트 화면과 필터 상태 조정
- `TaskManagementToolbar`: 검색과 상태·리셋 필터
- `AllowanceCard`: 수주권 표시와 편집
- `ManagedTaskGroup`: 그룹 접기와 결과 요약
- `ManagedTaskRow`: 리셋 정보와 활성 스위치
- `CustomTaskManager`: 기존 `CustomTaskList`의 기능을 관리 화면 문맥에 맞게 재사용 또는 이름을 정리
- `taskResetPresentation`: 리셋 라벨, 필터 분류, 다음 초기화 시각 계산을 담당하는 순수 도메인 유틸리티

`DefaultTaskManager`의 데이터 로직은 새 컴포넌트로 이동하고 기존 컴포넌트는 제거한다. 완료 체크용 `TaskItem`은 변경하지 않는다.

## 데이터 흐름

- 현재 캐릭터: `useCharacterStore`
- 기본 숙제 활성 상태: `useCurrentDisabledDefaultTaskIds`, `toggleDefaultTaskEnabled`
- 커스텀 숙제: 기존 `useTaskStore` 액션
- 완료 후 18시간 남은 시간: `completedAtByCharacter`
- 수주권: `useAllowanceStore`
- 필터와 검색: `TaskManagementPage`의 로컬 상태

새 영속 UI 상태나 데이터 마이그레이션은 추가하지 않는다.

## 시간 갱신

- 관리 화면은 현재 시각을 1분마다 갱신한다.
- 카운트다운은 분 단위까지만 표시하고 초 단위 애니메이션은 사용하지 않는다.
- 페이지가 백그라운드에서 돌아오면 다음 렌더에서 현재 시각을 다시 계산한다.
- 고정 시각 계산은 기존 KST 리셋 규칙 유틸리티를 확장해 사용한다.

## 반응형과 상호작용

- 390px: 모든 카드 한 열, 필터 칩 가로 스크롤
- 430px: 한 열 유지, 버튼 라벨이 잘리지 않도록 여백 확보
- 768px: 상단 도구 2열, 숙제 목록은 읽기 폭을 제한
- 긴 숙제 이름은 2줄까지 허용하고 메타데이터는 말줄임하지 않는다.
- 가로 페이지 스크롤은 발생하지 않아야 한다.
- 애니메이션은 그룹 펼침과 편집 전환에 짧은 opacity/transform만 사용한다.
- `prefers-reduced-motion`에서는 전환을 제거한다.

## 접근성

- 검색 필드는 명시적 레이블을 갖는다.
- 필터는 `aria-pressed`를 사용하는 버튼 그룹으로 구성한다.
- 그룹 버튼은 `aria-expanded`와 연결된 패널 ID를 갖는다.
- 활성/숨김은 상태가 음성으로 전달되는 실제 버튼 또는 체크박스를 사용한다.
- 아이콘 전용 버튼은 `aria-label`을 제공한다.
- 포커스 표시를 유지하고 키보드로 모든 기능을 사용할 수 있어야 한다.
- 수량 변경 결과는 `aria-live="polite"`로 전달한다.

## 오류와 빈 상태

- 활성 캐릭터가 없으면 기본 캐릭터 이름과 빈 설정으로 안전하게 렌더링한다.
- 수주권의 잘못된 입력은 0~100으로 정규화한다.
- 알 수 없는 리셋 규칙은 `수동 초기화`로 표시한다.
- 검색 결과 없음, 활성 숙제 없음, 숨긴 숙제 없음 상태를 구분한다.
- 커스텀 숙제 폼의 기존 제목 필수 검증을 유지한다.

## 테스트

### 단위 테스트

- 리셋 규칙별 라벨
- 다음 고정 초기화 시각과 날짜 경계
- 18시간 남은 시간
- 상태·초기화 필터 조합
- 검색 대상과 결과 수
- 수주권 입력 제한

### 통합 및 브라우저 검증

- `/tasks`에서 `/tasks/manage` 이동과 뒤로가기
- `/tasks/manage` 직접 진입
- 캐릭터 전환 후 활성/숨김 설정 분리
- 기본 숙제 활성/숨김 전환
- 커스텀 숙제 추가·수정·취소·삭제 회귀
- 수주권 수정과 새로고침 후 유지
- 390px, 430px, 768px 가로 넘침과 터치 영역
- 콘솔 오류 없음

### 최종 검증

- 전체 테스트
- TypeScript 프로덕션 빌드

## 제외 범위

- 숙제 체크 화면에 리셋 시각 상시 표시
- 사용자가 임의의 시각·요일을 만드는 고급 커스텀 규칙
- 알림 및 푸시
- 쿠로 수령일과 자동 만료
- History 통계 화면
