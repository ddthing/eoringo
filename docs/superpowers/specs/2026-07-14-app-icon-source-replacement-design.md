# 대표 앱 아이콘 원본 교체 설계

## 목표

현재 대표 아이콘 시스템의 원본과 모든 파생 아이콘을 사용자 제공 `photo_2026-07-14_12-17-50.jpg` 이미지로 교체한다. 아이콘 파일명과 웹 연결 경로는 유지해 애플리케이션 구조를 변경하지 않는다.

## 원본 교체

- 새 JPG 파일을 수정하지 않은 원본 그대로 `assets/brand/app-icon-source.jpg`에 보존한다.
- 기존 `assets/brand/app-icon-source.png`는 제거해 이전 이미지와 혼용되지 않게 한다.
- 저장된 JPG와 사용자 제공 파일의 SHA-256 해시가 일치해야 한다.

## 파생 아이콘

- `icon-512.png`: 새 이미지의 전체 구도를 사용한다.
- `icon-192.png`: 512px 아이콘과 같은 전체 구도를 사용한다.
- `apple-touch-icon.png`: 전체 구도를 180×180으로 축소한다.
- `icon-maskable-512.png`: 얼굴과 두 노란 눈이 중앙 안전 영역 안에 남도록 새 이미지를 축소하고 바깥 배경을 자연스럽게 확장한다.
- `favicon-32x32.png`, `favicon-16x16.png`: 새 이미지의 얼굴과 두 눈을 중심으로 확대 크롭한다.
- `favicon.ico`: 새 16px 및 32px 파비콘을 포함해 다시 생성한다.

## 연결 유지

- `index.html`의 favicon, Apple Touch Icon 및 manifest 링크는 변경하지 않는다.
- `site.webmanifest`의 아이콘 경로와 크기도 변경하지 않는다.
- 모든 기존 파생 파일을 같은 이름으로 교체해 이전 이미지가 남지 않게 한다.

## 범위 제외

- 앱 내부 로고와 헤더 UI 변경
- manifest 구조 변경
- 서비스 워커 및 캐시 버전 관리 추가
- 소셜 공유 이미지 추가

## 검증

- 원본 JPG의 해시 일치를 확인한다.
- 모든 PNG의 실제 픽셀 크기를 확인한다.
- 16px 및 32px 파비콘에서 얼굴과 두 눈이 식별되는지 확인한다.
- maskable 아이콘에서 얼굴이 안전 영역 밖으로 잘리지 않는지 확인한다.
- `.ico`가 16px 및 32px 두 항목을 포함하는지 확인한다.
- 아이콘과 manifest의 로컬 HTTP 경로가 모두 200으로 응답하는지 확인한다.
- 프로덕션 빌드 결과에 이전 원본이 포함되지 않는지 확인한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
