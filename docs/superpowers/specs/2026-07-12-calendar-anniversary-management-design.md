# 일정 탭 기념일 관리 복구 설계

## 목표

Home에 표시되는 기념일을 사용자가 일정 탭에서 추가하고 삭제할 수 있게 한다. 일정 탭은 날짜 데이터 관리, Home은 가까운 기념일 조회라는 역할을 유지한다.

## 범위

- 일정 탭 월간 달력 아래에 `기념일` 섹션 추가
- 현재 캐릭터의 기념일 목록 표시
- 기념일 추가 폼과 삭제 기능 제공
- 기존 `useDdayStore`, `getDdayLabel`, 확인 대화상자 재사용
- Home의 `UpcomingAnniversaryWidget` 동작 유지
- 기존 저장 데이터와 백업 형식 변경 없음

수정 기능, 반복 기념일, 알림, 달력 셀 마커는 이번 범위에 포함하지 않는다.

## 화면 구조

- 월간 달력 다음, 출처·범례 영역 이전에 배치한다.
- 헤더: `기념일` / `기념일 관리` / `추가` 버튼
- 목록 행: 제목, 날짜, D-day, 삭제 버튼
- 추가 버튼을 누르면 제목과 날짜 입력 폼이 펼쳐진다.
- 저장 또는 취소 후 폼을 닫는다.
- 등록된 항목이 없으면 짧은 빈 상태와 `기념일 추가` 행동을 제공한다.

## 동작

- 데이터는 현재 활성 캐릭터에만 추가한다.
- 제목 공백 또는 날짜 누락 시 저장하지 않고 입력 필드 근처에 오류 문구를 표시한다.
- 삭제는 기존 확인 대화상자를 사용한다.
- 캐릭터가 바뀌면 해당 캐릭터의 기념일 목록을 표시하고 열린 폼은 초기화한다.
- 날짜순으로 정렬하며 같은 날짜는 제목순으로 정렬한다.

## 컴포넌트

- 기존 미사용 `DdayCard`를 일정 화면 문맥에 맞게 `CalendarAnniversaryManager`로 정리한다.
- 저장·삭제 데이터 로직은 새로 복제하지 않고 기존 스토어 액션을 사용한다.
- CalendarPage는 배치만 담당한다.

## 접근성 및 반응형

- 모든 입력에 label 또는 aria-label 제공
- 추가·취소·삭제 버튼 최소 44px 터치 영역
- 오류 문구 `aria-live="polite"`
- 삭제 버튼에 기념일 제목을 포함한 aria-label 제공
- 390px과 430px은 한 열, 768px은 목록 행 정보만 가로 배치
- 가로 스크롤 없음
- 애니메이션은 짧은 opacity 전환만 사용하고 reduced-motion을 존중

## 테스트

- 빈 상태 렌더링
- 제목·날짜 검증
- 현재 캐릭터에 추가
- 캐릭터별 목록 분리
- 삭제 확인 흐름의 스토어 회귀
- Home 조회 위젯 데이터 연동 유지
- 390px, 430px, 768px 브라우저 검증
- 전체 test/build

## 완료 후 다음 작업

기념일 관리 검증과 커밋이 끝난 뒤 숙제 탭의 순서 변경 기능을 편집 모드로 이동한다.
