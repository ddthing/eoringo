# 숙제 탭·Home 오늘 체크 일관성 개선 설계

## 목표

- 오늘·주간 숙제 탭에서 사용하지 않는 빠른 추가 기능을 제거한다.
- 커스텀 숙제의 중복 자동 메타를 제거하되 사용자 설명과 메모는 유지한다.
- 숙제 탭에서 변경한 순서를 Home 오늘 체크에도 동일하게 반영한다.
- Home 오늘 체크에서 숨겨진 미완료 숙제를 펼쳐 바로 조작할 수 있게 한다.
- 다회 숙제 카운트 컨트롤을 Home과 숙제 탭에서 동일한 얇은 형태로 통일한다.

기존 숙제 완료 데이터, 초기화 규칙, 캐릭터별 활성화 설정, History 저장 형식은 변경하지 않는다.

## 확인된 원인

숙제 탭은 `useTaskUiStore.orderByGroup`을 읽어 캐릭터·기간·그룹별 저장 순서를 적용한다. Home의 `useHomeTodayTasks`와 `getHomeTodayTaskGroups`는 이 저장소를 읽지 않고 `priority`로 다시 정렬한다. 따라서 숙제 탭에서 순서를 변경해도 Home에는 반영되지 않는다.

## 기능 정리

### 빠른 추가 제거

- `/tasks`의 오늘 및 주간 화면에서 `숙제 빠르게 추가` 폼을 제거한다.
- 관련 `quickTitle` 상태, submit 핸들러, `addCustomTask` 구독과 아이콘 import도 제거한다.
- 커스텀 숙제 추가·수정은 기존 `/tasks/manage` 상세관리에서 계속 제공한다.

### 커스텀 숙제 자동 메타 제거

- 커스텀 숙제 행의 `수동 · 캐릭터별` 자동 메타를 표시하지 않는다.
- 사용자가 직접 입력한 설명 또는 메모는 기존처럼 제목 아래에 표시한다.
- 초기화 방식과 캐릭터 범위 데이터 자체는 변경하지 않는다.

## 공통 순서 계산

### 순서 규칙

- 기존 그룹 순서는 `무작위 임무 → 납품 → 전투/레이드 → PvP → 하우징 → 생활 → 이벤트/기타 → 커스텀`을 유지한다.
- 그룹 내부에서는 `orderByGroup[characterId:view:group]`에 저장된 순서를 우선한다.
- 저장 목록에 없는 새 숙제는 기존 `priority` 순서로 저장 숙제 뒤에 배치한다.
- 삭제되었거나 현재 필터에 없는 ID는 무시한다.
- 캐릭터와 `daily`·`weekly` 범위는 기존 scope key로 격리한다.

### 공통 유틸리티

순서 계산을 순수 함수로 분리해 숙제 탭과 Home이 함께 사용한다. 숙제 탭은 그룹별 렌더링에 사용하고, Home은 일일·주간 카테고리의 미완료 항목을 선택하기 전에 같은 함수를 사용한다.

Home의 `기타(custom)` 카테고리는 숙제 탭에 독립 순서 편집 화면이 없으므로 기존 `priority` 순서를 유지한다.

### 완료 항목 처리

Home 오늘 체크는 기존처럼 완료 항목을 목록에서 제외한다. 미완료 항목은 숙제 탭에서 저장한 상대 순서를 유지한다. 완료 처리로 항목이 사라져도 나머지 순서는 변경되지 않는다.

## 오늘 체크 펼침

- 각 일일·주간·기타 카테고리는 기본적으로 미완료 숙제 두 개를 표시한다.
- 추가 미완료 항목이 있으면 카드 하단에 `미완료 n개 모두 펼치기` 버튼을 표시한다.
- 버튼을 누르면 해당 카테고리의 미완료 항목을 모두 표시한다.
- 펼친 상태의 같은 위치에는 `접기` 버튼을 표시한다.
- `aria-expanded`와 명확한 접근성 이름을 제공한다.
- 펼침 상태는 `HomeTodayCheck`의 로컬 UI 상태이며 저장하지 않는다. 화면 재진입 및 캐릭터 변경 시 기본 접힘 상태로 돌아간다.
- 전체 완료 및 빈 카테고리의 기존 표현은 유지한다.

## 공통 다회 카운트 컨트롤

`TaskCountStepper`를 공통 컴포넌트로 분리한다.

- 사용처: `HomeTodayCheckItem`, 숙제 탭의 `TaskItem`
- 표시: 얇은 `− 0/12 +` 인라인 구성
- 외곽의 두꺼운 캡슐 테두리와 별도 배경은 제거한다.
- 감소·증가 버튼은 시각적으로 가볍지만 실제 터치 영역은 각각 최소 44px을 유지한다.
- 현재값은 tabular number로 표시한다.
- 최솟값과 최댓값에서 해당 버튼을 비활성화한다.
- 기존 `onSetCount`와 완료 판정 로직을 그대로 사용한다.

## 컴포넌트 변경

- `TaskManagerPage`: 빠른 추가 제거, 공통 정렬 함수 사용
- `useHomeTodayTasks`: `useTaskUiStore.orderByGroup` 구독 및 공통 정렬 적용
- `getHomeTodayTaskGroups`: 전달받은 작업 순서를 보존하고 펼침에 필요한 전체 미완료 목록 제공
- `HomeTodayCheck`: 카테고리별 펼침 상태와 disclosure 버튼 관리
- `HomeTodayCheckItem`: 공통 `TaskCountStepper` 사용
- `TaskItem`: 공통 `TaskCountStepper` 사용
- `CustomTaskList`: `showMeta`를 제거해 자동 메타를 숨김

## 반응형 및 상호작용

- 390px에서 제목, 그룹 태그, 카운트 컨트롤이 겹치지 않아야 한다.
- 긴 제목은 기존 말줄임을 유지한다.
- 펼침 버튼은 카드 전체 너비와 최소 44px 높이를 사용한다.
- 펼침 화살표 회전 외의 새 애니메이션은 추가하지 않는다.
- hover, active, focus-visible 상태는 기존 디자인 토큰을 사용한다.

## 오류 및 예외 처리

- 저장 순서가 비었거나 손상되면 `priority` 순서로 안전하게 대체한다.
- 저장 순서에 중복 ID가 있으면 첫 번째 위치만 사용한다.
- 현재 카테고리와 맞지 않는 ID는 무시한다.
- 미완료 항목이 두 개 이하가 되면 펼침 버튼을 제거한다.
- 펼친 상태에서 완료 처리로 항목 수가 줄어도 남은 항목을 정상 표시한다.

## 테스트

### 단위 테스트

- 저장 순서 우선 적용
- 저장되지 않은 새 숙제 뒤쪽 배치
- 삭제·중복·다른 카테고리 ID 무시
- 캐릭터·daily·weekly scope 분리
- Home 기본 두 개 표시와 전체 미완료 목록
- 완료 항목 제외 후 순서 유지
- 카운트 최솟값·최댓값 처리 회귀

### 브라우저 검증

- 오늘·주간에서 빠른 추가 영역이 제거되는지 확인한다.
- 커스텀 숙제의 자동 메타는 사라지고 사용자 메모는 유지되는지 확인한다.
- 숙제 탭에서 순서를 변경한 뒤 Home에 같은 순서가 반영되는지 확인한다.
- Home의 카테고리별 전체 펼침과 접기를 확인한다.
- Home과 숙제 탭에서 단골 거래 카운터가 동일한 형태와 동작을 갖는지 확인한다.
- 390px에서 가로 넘침과 터치 영역을 확인한다.
- 콘솔 오류가 없는지 확인한다.

### 최종 검증

- 전체 `test`
- TypeScript 및 Vite `build`

## 제외 범위

- Home에서 숙제 순서를 직접 편집하는 기능
- `기타(custom)` 전용 순서 편집 화면
- 커스텀 숙제 데이터 구조 또는 초기화 규칙 변경
- 펼침 상태의 영구 저장
- 숙제 완료·초기화·History 로직 변경
