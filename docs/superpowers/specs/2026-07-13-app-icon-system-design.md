# 대표 앱 아이콘 시스템 설계

## 목표

제공된 캐릭터 이미지를 앱의 대표 이미지로 사용한다. 앱 설치 아이콘에서는 원본의 전체 구도를 유지하고, 매우 작은 파비콘에서는 얼굴과 두 노란 눈을 강조해 크기별 식별성을 확보한다.

## 원본 관리

- 사용자 제공 PNG를 브랜드 원본으로 저장소에 보존한다.
- 원본은 빌드 산출물에 불필요하게 복사되지 않는 별도 브랜드 자산 경로에 둔다.
- 파생 아이콘은 원본의 색상과 캐릭터 형태를 바꾸지 않고 크롭, 축소 및 안전 여백만 적용한다.

## 파생 아이콘

### 표준 앱 아이콘

- `512×512`: 현재의 전체 캐릭터 구도를 유지한다.
- `192×192`: 512 아이콘과 동일한 구도를 사용한다.
- `180×180`: iOS Apple Touch Icon으로 사용한다.
- 운영체제가 아이콘 형태를 마스킹하므로 이미지 자체에 둥근 모서리를 넣지 않는다.

### Maskable 아이콘

- `512×512` 크기로 생성한다.
- 캐릭터의 얼굴과 두 눈이 중앙 안전 영역 안에 들어오도록 전체 이미지를 축소해 여백을 확보한다.
- 바깥 여백은 원본의 연한 파란 배경과 자연스럽게 이어지도록 처리한다.
- Android의 원형, 둥근 사각형 및 기타 마스크에서도 핵심 얼굴이 잘리지 않아야 한다.

### 작은 파비콘

- `32×32`, `16×16` PNG와 다중 크기 `.ico`를 생성한다.
- 얼굴과 두 노란 눈을 중심으로 확대 크롭한다.
- 검과 망토 전체를 억지로 포함하지 않고 작은 크기에서 인식되는 실루엣을 우선한다.

## 파일 구조

- 브랜드 원본: `assets/brand/app-icon-source.png`
- 웹 아이콘: `public/icons/`
- manifest: `public/site.webmanifest`
- 기본 `.ico`: `public/favicon.ico`

웹 아이콘 파일은 다음 이름을 사용한다.

- `icon-192.png`
- `icon-512.png`
- `icon-maskable-512.png`
- `apple-touch-icon.png`
- `favicon-32x32.png`
- `favicon-16x16.png`

## 웹 메타데이터

`index.html`에 다음 항목을 연결한다.

- 기본 `favicon.ico`
- 16px 및 32px PNG favicon
- Apple Touch Icon
- `site.webmanifest`
- 앱 배경과 일치하는 `theme-color`

`site.webmanifest`에는 다음 값을 정의한다.

- 앱 이름과 짧은 이름
- `/` 시작 URL과 scope
- `standalone` 표시 방식
- 앱 배경색과 테마색
- 192px, 512px 및 maskable 아이콘

## 범위 제외

- 서비스 워커
- 오프라인 캐시
- 푸시 알림
- 소셜 공유용 Open Graph 대표 이미지
- 앱 내부 로고나 헤더 UI 변경

## 성능

- 브라우저가 실제로 요청하는 파생 아이콘만 `public`에 둔다.
- 큰 원본 이미지는 `public` 밖에 보존해 배포 산출물에 포함되지 않게 한다.
- 작은 아이콘은 시각 품질을 유지하는 범위에서 PNG 용량을 최적화한다.

## 검증

- 각 PNG의 실제 픽셀 크기와 정사각형 비율을 확인한다.
- 작은 파비콘에서 두 눈과 얼굴이 식별되는지 확인한다.
- maskable 안전 영역에서 얼굴이 잘리지 않는지 확인한다.
- manifest JSON과 모든 아이콘 경로가 유효한지 확인한다.
- 로컬 브라우저에서 favicon, Apple Touch Icon 및 manifest가 HTTP 200으로 제공되는지 확인한다.
- 프로덕션 빌드 결과에 필요한 파일만 포함되는지 확인한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
