# 전장 표기·기념일 편집·숙제 정보·앱 크레딧 개선 설계

## 목표

- 전장 범례와 요약의 정식 명칭을 동일한 형식으로 표시한다.
- Home과 일정에서 기념일을 추가·수정하고 동일 데이터가 즉시 동기화되게 한다.
- 집사와 총사령부 소대 임무의 명칭과 안내를 실제 파견 흐름에 맞춘다.
- 앱 정보에 크레딧과 문의 링크를 제공한다.
- 기존 사용자 데이터, 캐릭터별 범위, History 형식과 화면별 역할은 유지한다.

## 변경 단위

### 전장 정식 표기

`frontlineMaps`의 `displayName`을 다음 값으로 통일한다.

- `봉인된 바위섬 (쟁탈전)`
- `영광의 평원 (쇄빙전)`
- `온살 하카이르 (계절 끝 합전)`
- `워코 치테 (연습전)`
- `외곽 유적지대 (제압전)`

달력 칸과 작은 배지에서 사용하는 `봉바`, `쇄빙`, `온살`, `워코`, `제압` 약칭은 유지한다. 범례, 오늘 전장, 내일 전장과 Home 전장 요약처럼 `displayName`을 사용하는 위치에는 같은 정식 표기가 나타난다. 로테이션 계산과 맵 ID는 변경하지 않는다.

### 기념일 수정과 동기화

기념일 저장소에 `updateEvent(characterId, eventId, patch)` 액션을 추가한다. 수정 시 기존 이벤트 ID와 `characterId`를 보존하고 제목과 날짜만 교체한 뒤 날짜순으로 다시 정렬한다. 이벤트가 없거나 다른 캐릭터에 속하면 아무 데이터도 만들지 않는다.

Home의 `UpcomingAnniversaryWidget`과 일정의 `CalendarAnniversaryManager`는 각 화면의 기존 입력 폼을 추가·수정 겸용으로 사용한다.

- 목록 항목에 44px 이상 편집 버튼을 제공한다.
- 편집을 시작하면 기존 제목과 날짜를 폼에 채운다.
- 저장 버튼 문구는 추가와 수정 상태를 구분한다.
- 수정 저장은 `updateEvent`, 신규 저장은 `addEvent`를 호출한다.
- 취소·닫기·캐릭터 전환 시 편집 ID와 초안을 함께 초기화한다.
- Home에서는 현재 노출 중인 최대 세 개 기념일을 바로 수정할 수 있다.
- 일정에서는 현재 캐릭터의 전체 기념일을 수정할 수 있다.

두 화면은 동일한 Zustand 이벤트 배열을 구독하므로 한 화면의 수정 결과가 다른 화면에 즉시 반영된다. 별도 복제 데이터나 화면 간 동기화 이벤트는 만들지 않는다. 기존 삭제 확인 팝업과 날짜 검증은 유지한다.

### 집사 수행 안내

`집사 수행 확인`의 18시간 규칙과 기존 저장 동작은 유지한다. 사용자에게 보이는 설명은 완료 기준이 아니라 파견 기준임을 명확히 하여 `파견 후 18시간 후 확인할 수 있습니다.`로 변경한다.

### 총사령부 소대 임무

- `총사령부 소대 일반 임무` 기본 숙제를 목록에서 제거한다.
- `총사령부 소대 중요 임무`를 `총사령부 소대 특수 임무 확인`으로 변경한다.
- 특수 임무는 기존 주간 체크 방식과 화요일 17:00 초기화를 유지한다.
- 설명에 `파견 후 18시간 후 확인`, `동일한 임무는 화요일 17:00 초기화`, `파견 실패 시 즉시 재파견 가능`을 안내한다.

별도 파견 버튼, 파견 시각, 임무 종류, 성공·실패 상태는 추가하지 않는다. 제거된 일반 임무의 과거 완료·비활성 값이 저장소에 남아 있어도 화면과 진행률 계산에서는 기본 템플릿에 없으므로 무시된다. 이번 변경을 위한 저장소 버전 상승이나 사용자 데이터 삭제는 하지 않는다.

### 앱 정보와 문의

설정의 앱 정보 카드에 다음 크레딧을 별도 줄로 표시한다.

- `Thanks to ADD, 꼭짓점, 미여워, KILL`
- `Special thanks to 루피`

같은 카드에 `문의하기` 링크를 추가하고 `https://coner.luv3r.me/`를 새 탭에서 연다. 링크에는 외부 링크 아이콘과 명확한 접근성 이름을 제공하고 `rel="noreferrer"`를 사용한다. 기존 앱 소개와 로컬 데이터 안내는 유지한다.

## 오류 처리와 데이터 보존

- 기념일 폼은 기존 제목 필수·날짜 유효성 검증을 추가와 수정에 동일하게 적용한다.
- 수정 대상이 저장 전에 삭제됐거나 캐릭터가 바뀌면 잘못된 새 이벤트를 생성하지 않는다.
- 전장 맵 ID, 기념일 데이터 형식, 숙제 완료 기록 형식은 변경하지 않는다.
- 문의 링크가 외부에서 응답하지 않더라도 앱 내부 상태에는 영향을 주지 않는다.

## 검증

자동 검증:

- 다섯 전장의 `displayName`이 정확한지 확인한다.
- 기존 전장 로테이션과 약칭이 유지되는지 확인한다.
- 기념일 수정이 ID와 캐릭터를 보존하고 날짜순으로 재정렬하는지 확인한다.
- 다른 캐릭터의 이벤트나 없는 ID를 수정하지 않는지 확인한다.
- 일반 소대 임무가 기본 목록에서 제거됐는지 확인한다.
- 집사와 특수 소대 임무의 제목, 설명, 초기화 규칙이 정확한지 확인한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.

브라우저 검증:

- Home과 일정에서 기념일 추가·수정·취소·캐릭터 전환을 확인한다.
- 한 화면에서 수정한 기념일이 다른 화면에 즉시 나타나는지 확인한다.
- 일정 범례와 오늘/내일 전장 정식 표기를 확인한다.
- 숙제 탭과 상세관리에서 집사·소대 항목과 안내를 확인한다.
- 설정 앱 정보의 크레딧과 문의 링크를 확인한다.
- 모바일 폭에서 편집 버튼과 폼이 겹치지 않고 콘솔 오류가 없는지 확인한다.

## 제외 범위

- 소대 파견 타이머, 임무 종류, 성공·실패 상태의 별도 추적
- Home에 기념일 전체 목록 또는 삭제 기능 추가
- 전장 로테이션 패턴 변경
- 외부 문의 사이트의 내용이나 가용성 관리
