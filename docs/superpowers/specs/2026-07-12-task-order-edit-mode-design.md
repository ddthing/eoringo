# 숙제 순서 편집 모드 설계

## 목표

매일 사용하는 숙제 체크 화면에서 순서 변경 핸들을 숨겨 조작 밀도를 낮추고, 사용자가 명시적으로 편집 모드에 들어갔을 때만 재정렬할 수 있게 한다.

## 동작

- 숙제 검색 아래 도구 영역에 `순서 편집` 버튼을 배치한다.
- 평상시에는 드래그 핸들을 렌더링하지 않는다.
- 편집 모드에서는 버튼이 `편집 완료`로 바뀌고 각 숙제에 핸들이 나타난다.
- 편집 중 숙제 체크, 카운트 증감, 빠른 추가, 일일/주간 전환, 캐릭터 전환을 잠가 오작동을 막는다.
- 그룹 접기와 순서 변경은 계속 사용할 수 있다.
- 편집 모드를 종료하면 기존 저장 순서가 즉시 반영된다.
- 페이지 이동 또는 캐릭터·기간 변경 시 편집 모드를 자동 종료한다.

## 입력 방식

- 데스크톱: 기존 HTML drag-and-drop 유지
- 키보드: 핸들에 포커스 후 위/아래 화살표로 이동
- 모바일: 위로/아래로 버튼을 핸들 옆에 제공해 네이티브 드래그 의존을 제거
- 이동 불가능한 첫/마지막 버튼은 disabled 처리
- 변경 결과는 `aria-live="polite"`로 알린다.

## 시각 및 접근성

- `순서 편집`, `편집 완료`, 위/아래 버튼 모두 최소 44px
- 편집 모드는 얇은 강조 배너와 핸들 색상으로 구분
- 색상만으로 상태를 전달하지 않고 `순서 편집 중` 문구 표시
- reduced-motion에서는 위치 전환 애니메이션 제거
- 일반 모드의 숙제 행 높이와 체크 UI는 변경하지 않는다.

## 구조

- `TaskManagerPage`는 `isOrderEditing` 상태와 모드 전환만 관리한다.
- `TaskOrderControls`가 드래그·키보드·모바일 이동 버튼을 담당한다.
- 기존 `reorderTaskIds`, `setGroupOrder`를 재사용한다.
- 영속 데이터와 마이그레이션은 변경하지 않는다.

## 테스트

- 기본 모드에서 핸들 없음
- 편집 모드 진입·종료
- 체크 및 카운트 입력 잠금
- 위/아래 이동과 경계 disabled
- 캐릭터·기간 변경 시 자동 종료
- 기존 순서 저장 회귀
- 390/430/768px 터치 영역과 가로 넘침
- 전체 test/build

## 제외 범위

- 그룹 자체 순서 변경
- 다른 그룹으로 숙제 이동
- 사용자 정의 그룹 생성
