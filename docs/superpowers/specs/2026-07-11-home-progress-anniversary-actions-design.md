# Home 진행도·기념일 및 일정 화면 정리 설계

## 목표

- 오늘 체크와 숙제 진행도 사이의 중복 이동 동작을 제거한다.
- 숙제 진행도의 일일·주간·기타 통계를 구분선 없이 가볍게 읽히도록 정리한다.
- Home에서 기념일을 확인할 뿐 아니라 바로 추가할 수 있게 한다.
- 일정 화면에서 월간 달력과 중복되는 7일 요약을 제거한다.
- 기존 숙제 및 기념일 데이터 구조와 동작은 유지한다.

## 숙제 진행도

- `HomeProgress` 헤더의 `전체 숙제 보기` 버튼을 제거한다.
- 숙제 화면 이동은 바로 위 `오늘 체크` 카드의 버튼 한 곳에서만 제공한다.
- 일일·주간·기타 행 사이의 `divide-y` 구분선을 제거한다.
- 각 행의 라벨, 완료 수, 백분율, 진행 막대는 유지한다.
- 도넛 그래프와 진행 애니메이션은 변경하지 않는다.

## 기념일 추가

- `UpcomingAnniversaryWidget` 헤더 오른쪽에 `기록` 버튼을 추가한다.
- 버튼을 누르면 같은 카드 안에서 기념일 이름과 날짜 입력 폼을 펼친다.
- 열린 상태에서는 버튼을 `닫기`로 표시한다.
- 이름과 날짜가 모두 입력되어야 추가할 수 있다.
- 추가 시 현재 활성 캐릭터의 기념일로 기존 `useDdayStore.addEvent`를 호출한다.
- 추가 완료 후 입력값을 비우고 폼을 닫는다.
- 빈 상태와 등록된 기념일 목록 모두 같은 추가 진입점을 제공한다.
- 기존 목록 정렬과 최대 3개 표시 규칙은 유지한다.

## 컴포넌트 구조

- 입력 폼과 상태는 `UpcomingAnniversaryWidget` 내부에 둔다.
- 기존에 연결되지 않은 `DdayCard`의 검증된 입력 흐름을 재사용하되 별도 관리 화면은 만들지 않는다.
- 저장소와 이벤트 타입은 변경하지 않는다.

## 일정 화면

- `CalendarPage`의 `7일 일정` 제목, `오늘부터 7일` 안내, 7개 날짜 카드 영역을 일괄 제거한다.
- 해당 영역만을 위한 `nextSevenDays` 계산과 `CalendarDays` 아이콘 import를 제거한다.
- 오늘 요약, 월간 전장/하우징 달력, 하우징 매물 메모, 범례는 유지한다.
- 전장 및 하우징 계산 도메인 로직은 변경하지 않는다.

## 접근성 및 반응형

- 기록/닫기 버튼은 기존 `secondary-button`을 사용해 충분한 터치 영역을 유지한다.
- 입력에는 `기념일 이름`, `기념일 날짜` 접근성 이름을 제공한다.
- 390px에서 헤더, 폼, 목록의 가로 넘침이 없어야 한다.

## 검증

- Home 진행도 카드에 중복 이동 버튼과 행 구분선이 없는지 확인한다.
- 빈 상태와 목록 상태에서 기록 폼을 열 수 있는지 확인한다.
- 기념일을 추가하면 즉시 목록에 표시되는지 확인하고 검증 데이터는 제거한다.
- 일정 화면에서 7일 일정 영역이 사라지고 나머지 영역 순서와 동작이 유지되는지 확인한다.
- 전체 test와 production build를 실행한다.
