# Settings 데이터 및 오늘 체크 밀도 개선 설계

## 목표

- Settings의 백업·복원과 초기화 기능을 하나의 명확한 데이터 영역으로 묶는다.
- 오늘 체크의 다회 숙제 조작부가 다른 항목보다 과도하게 무거워 보이지 않게 한다.
- 오늘 체크에서 숙제 제목과 중복되는 그룹 태그를 제거한다.
- 저장소, 데이터 형식, 숙제 완료 동작은 변경하지 않는다.

## Settings 데이터 구조

- 기존 `백업 및 복원` 카드와 `데이터 관리` 카드를 하나의 `데이터` 카드로 통합한다.
- 카드 상단에는 `데이터` 제목과 브라우저 로컬 저장 안내를 한 번만 표시한다.
- 첫 번째 내부 영역은 `백업 및 복원`으로 구성하고 기존 백업·복원 버튼과 상태 메시지를 유지한다.
- 두 번째 내부 영역은 얇은 구분선 아래 `데이터 초기화`로 구성한다.
- 초기화 설명, 확인 다이얼로그, 위험 색상, 처리 상태는 유지한다.
- `local data` 영문 라벨의 반복 표시는 제거한다.

## Settings 앵커 호환

- `#backup`은 통합 카드 상단으로 이동한다.
- `#data`는 통합 카드 내부의 `데이터 초기화` 영역으로 이동한다.
- 기존 북마크와 내부 링크 모두 404나 잘못된 위치 이동 없이 작동해야 한다.
- Settings 표시 순서는 `내 캐릭터 → 테마 → 알림 → 데이터 → 앱 정보`로 단순화한다.

## 오늘 체크 그룹 태그

- `HomeTodayCheckItem`의 작은 그룹 태그를 모든 숙제에서 제거한다.
- `무작위 임무: 숙련자` 아래에 반복되던 `무작위 임무`도 제거된다.
- 숙제 제목, 일일·주간·기타 섹션, 완료 수는 유지한다.

## 다회 숙제 컨트롤

- 단골 거래 같은 다회 숙제의 `− 0/12 +` 기능은 유지한다.
- 컨트롤 전체를 감싸는 큰 캡슐 테두리와 배경을 제거한다.
- 감소·증가 버튼은 최소 44px 터치 영역을 유지한다.
- 현재 횟수만 작은 중립 배경의 숫자 배지로 표시한다.
- 다회 숙제 행의 높이는 단일 체크 숙제와 최대한 비슷하게 맞춘다.
- 비활성 상태와 접근성 이름은 기존 동작을 유지한다.

## 컴포넌트 구조

- `BackupRestorePanel`과 `DataManagementPanel`의 동작은 하나의 `DataSettingsPanel` 안에서 재사용하거나 내부 하위 영역으로 재구성한다.
- `SettingsPage`는 통합 데이터 패널을 한 번만 렌더링한다.
- `HomeTodayCheckItem`은 제목과 조작부만 렌더링한다.
- Zustand 저장소와 도메인 로직은 변경하지 않는다.

## 검증

- Settings에 `local data` 반복 라벨이 없고 데이터 카드가 하나인지 확인한다.
- `#backup`과 `#data`가 각각 통합 카드 상단과 초기화 영역으로 이동하는지 확인한다.
- 오늘 체크에 그룹 태그가 없고 단골 거래 컨트롤의 터치 영역이 44px 이상인지 확인한다.
- 다회 횟수 증감이 기존처럼 동작하는지 확인하고 검증 데이터는 원상 복구한다.
- 390px에서 가로 넘침과 텍스트 잘림을 확인한다.
- 전체 test와 production build를 실행한다.
