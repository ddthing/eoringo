# 터치 영역 44px 통일 설계

## 목표

모바일에서 자주 누르는 버튼과 입력의 최소 터치 영역을 44×44px로 통일해 오조작을 줄인다. 정보 구조, 기능, 저장 데이터는 변경하지 않는다.

## 대상

- 숙제 탭: 일일/주간, 검색, 빠른 추가 입력·버튼, 상세 관리 링크
- 숙제 상세 관리: 상태·리셋 필터, 수주권 조절, 기본 숙제 활성 토글
- 일정 탭: 전장/하우징 월간, 범례, 외부 시트 링크
- 설정 탭: 캐릭터 추가, 수정, 대표 별, 삭제, 테마 버튼

Home은 감사에서 44px 미만 요소가 발견되지 않아 이번 범위에서 제외한다.

## 원칙

- 시각 아이콘 크기는 유지하고 투명 패딩으로 터치 영역만 확장한다.
- 행 높이가 불필요하게 커지지 않도록 인접 버튼은 44px 정사각형을 기본으로 한다.
- 텍스트 버튼은 `min-height: 44px`를 사용한다.
- 입력은 실제 input 또는 이를 감싸는 label 전체가 44px 이상이어야 한다.
- disabled 상태에서도 레이아웃 크기는 유지한다.
- focus-visible, hover, active 상태를 기존 테마와 일치시킨다.
- 설정의 수정·대표·삭제는 동일한 44px 크기로 통일한다.

## 검증

- 390px에서 각 대상의 bounding box 측정
- 430px과 768px 가로 넘침 없음
- 캐릭터 수정·대표·삭제 기능 회귀
- 숙제 검색·추가·기간 전환 회귀
- 일정 월간 전환·범례 링크 회귀
- 전체 test/build

## 제외 범위

- 카드 높이 재설계
- 색상과 타이포그래피 변경
- 데스크톱 전용 hover 효과 추가
