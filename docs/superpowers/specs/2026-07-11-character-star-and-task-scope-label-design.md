# 대표 캐릭터 별 및 숙제 범위 표기 정리 설계

## 목표

- 대표 캐릭터 상태를 모든 화면에서 동일한 별 표현으로 통일한다.
- 설정의 캐릭터 관리에서 중복되는 `대표` 글자 배지를 제거한다.
- 숙제 상세 관리에서 이미 상단 설명으로 전달되는 `캐릭터별` 문구를 반복하지 않는다.
- 대표 변경, 캐릭터 삭제, 숙제 설정 등 기존 동작은 변경하지 않는다.

## 대표 캐릭터 표현

- 대표 상태의 공통 시각 언어는 강조색으로 채워진 별이다.
- 비대표 상태는 채우지 않은 별을 사용한다.
- 설정의 각 캐릭터 행에서는 대표 상태를 캐릭터 이름 옆 글자 배지로 표시하지 않는다.
- 대표 설정 별 버튼은 삭제 버튼 옆에 유지한다.
- 대표 캐릭터의 별 버튼도 화면에 표시하되 채워진 상태로 나타낸다. 기존과 같이 이미 대표인 캐릭터에는 대표 변경 동작이 발생하지 않는다.
- 접근성 이름은 화면에 보이지 않는 `대표 캐릭터` 또는 `<이름> 대표 설정` 형태로 유지한다.

## 숙제 상세 관리 표기

- 일일 및 주간 상세 관리 목록에서 `캐릭터별` 보조 문구를 제거한다.
- 숙제 제목, 그룹, 완료 상태, 순서 변경, 표시 설정은 그대로 유지한다.
- 실제로 공통 범위인 항목의 `공통` 표기는 범위를 구분하는 정보이므로 유지한다.
- 상단의 현재 캐릭터 적용 안내 문구는 유지한다.

## 컴포넌트 구조

- `CharacterMainBadge`가 대표 상태의 채워진 별 표현을 담당한다.
- `CharacterManager`는 별 버튼에 대표 여부를 전달하고 별과 삭제 버튼의 배치를 담당한다.
- 숙제 범위 보조 문구는 기존 `TaskItem`/상세 관리 렌더링 경로에서 캐릭터 범위에 한해 숨긴다.
- 상태 저장소 및 데이터 모델은 변경하지 않는다.

## 검증

- 대표/비대표 캐릭터의 별 채움 상태를 컴포넌트 또는 브라우저에서 확인한다.
- 대표 변경과 삭제 버튼이 기존대로 동작하는지 확인한다.
- 일일·주간 상세 관리에서 `캐릭터별` 문구가 사라지고 `공통` 표기는 필요한 경우 유지되는지 확인한다.
- 전체 test와 production build를 실행한다.
- 검증 통과 후 Git 저장소를 초기화하고 `https://github.com/ddthing/eoringo.git`에 push한다.
