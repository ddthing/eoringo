# 숙제 상세관리 렌더 오류 및 화면 복구 설계

## 목표

- 완료시각 기록이 없는 새 사용자도 `/tasks/manage` 상세관리 화면에 정상 진입한다.
- 한 화면에서 렌더 오류가 발생해도 하단 탭으로 다른 경로에 이동하면 정상 화면으로 복구된다.
- 숙제 완료 데이터, 표시 설정, 초기화 규칙 및 저장 형식은 변경하지 않는다.

## 확인된 원인

`DefaultTaskManager`의 Zustand selector가 현재 캐릭터의 완료시각 기록이 없을 때 `?? {}`로 새 객체를 반환한다. 동일한 store snapshot에도 selector 결과 참조가 계속 바뀌므로 React의 외부 store 구독이 무한 갱신되고 `Maximum update depth exceeded` 오류가 발생한다.

전역 `ErrorBoundary`는 `Outlet`을 감싼 채 경로 변경 후에도 같은 인스턴스로 유지된다. 한 번 오류 상태가 되면 하단 탭으로 URL이 바뀌어도 오류 상태가 남아 새 화면을 렌더링하지 않는다.

## 수정 설계

### 안정적인 완료시각 selector

- 모듈 범위에 읽기 전용 빈 완료시각 객체를 한 번만 만든다.
- 현재 캐릭터의 완료시각 기록이 없으면 이 객체를 반환한다.
- selector를 순수 함수로 분리해 동일한 입력에 동일한 참조를 반환하는지 단위 테스트한다.
- 실제 완료시각 기록이 있으면 기존 캐릭터별 객체를 그대로 반환한다.

### 경로 변경 시 오류 복구

- 현재 `location.pathname`을 `ErrorBoundary`의 React `key`로 사용한다.
- 같은 경로 안에서는 기존 오류 화면과 새로고침 동작을 유지한다.
- 하단 탭 등으로 다른 경로에 이동하면 boundary가 새로 마운트되어 해당 화면을 정상 렌더링한다.
- URL 변경이나 하단 내비게이션 구조 자체는 바꾸지 않는다.

## 오류 처리

- 상세관리의 근본 렌더 오류는 안정적인 selector로 제거한다.
- 예상하지 못한 렌더 오류는 계속 `ErrorBoundary`에서 기록하고 안내한다.
- 오류가 난 경로로 다시 돌아왔을 때 원인이 남아 있으면 다시 오류 화면을 표시하며, 오류를 숨기거나 반복 재시도하지 않는다.

## 검증

- 완료시각 기록이 없는 store 상태에서 selector 결과가 연속 호출 간 같은 참조인지 검증한다.
- 완료시각 기록이 있는 상태에서는 해당 캐릭터의 기존 객체를 반환하는지 검증한다.
- 전체 단위 테스트와 프로덕션 빌드를 실행한다.
- 빈 상태에서 `숙제 → 상세 관리`로 이동해 화면이 열리고 콘솔에 무한 갱신 오류가 없는지 브라우저에서 확인한다.
- 의도적으로 오류 상태가 된 boundary에서 다른 경로로 이동할 때 정상 화면으로 복구되는 동작을 회귀 검증한다.

## 제외 범위

- 다른 Zustand selector의 일괄 리팩터링
- 상세관리 UI 또는 숙제 데이터 구조 변경
- 오류 화면 디자인 변경
