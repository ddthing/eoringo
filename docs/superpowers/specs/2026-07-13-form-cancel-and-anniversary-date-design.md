# 폼 취소 동작 및 기념일 날짜 입력 수정 설계

## 목표

- 기념일 날짜의 연도를 정확히 네 자리로 제한한다.
- 임시 입력 폼의 취소 또는 닫기 동작을 `폼 닫기 + 미저장 초안 폐기`로 통일한다.
- 기존 저장 데이터, 화면 구조, 네이티브 날짜 선택 UI는 유지한다.
- 현재 숙제 상세관리 화면을 중단시키는 컴파일 오류를 먼저 복구한다.

## 확인된 원인

1. `CustomTaskList`의 취소 버튼은 초안만 초기화하고 `isFormOpen`을 변경하지 않아 빈 폼이 계속 표시된다.
2. 기념일 입력 세 곳은 네이티브 `type="date"`에만 의존한다. 브라우저가 다섯 자리 이상의 연도를 입력값으로 받아도 애플리케이션에서 제한하거나 검증하지 않는다.
3. 일정 기념일 취소는 폼만 닫고 제목과 날짜를 유지해 다시 열었을 때 폐기한 초안이 나타난다.
4. 직전 커스텀 숙제 필터 연결부의 괄호 누락으로 Vite가 `CustomTaskList.tsx`를 컴파일하지 못한다.

## 날짜 입력 설계

### 공통 규칙

- 저장 형식은 기존과 같은 `YYYY-MM-DD` 문자열이다.
- 허용 범위는 `1000-01-01`부터 `9999-12-31`까지다.
- 연도는 네 자리, 월과 일은 각각 두 자리여야 한다.
- 달력상 존재하지 않는 날짜는 유효하지 않다.
- 기존 `type="date"`와 네이티브 날짜 선택 UI를 유지한다.

### 공통 컴포넌트

`AnniversaryDateField`를 추가해 다음 화면에서 재사용한다.

- 일정 탭 `CalendarAnniversaryManager`
- Home `UpcomingAnniversaryWidget`
- 기존 `DdayCard`

컴포넌트는 `value`, `onChange`, `name`, `aria-label`, 스타일 속성을 받는다. `min="1000-01-01"`, `max="9999-12-31"`를 제공하고, 입력 이벤트에서는 네 자리보다 긴 연도가 포함된 값을 상태에 반영하지 않는다.

### 검증

`anniversaryManagement` 도메인에 날짜 검증을 둔다. 빈 값과 잘못된 형식·범위·실재하지 않는 날짜를 구분해 저장을 차단한다. 세 입력 화면은 같은 검증 규칙을 사용해 UI마다 결과가 달라지지 않게 한다.

## 취소 및 닫기 설계

임시 초안을 가진 폼의 dismiss 동작은 아래 순서를 따른다.

1. 초안과 필드 오류를 초깃값으로 되돌린다.
2. 추가·수정 식별 상태를 해제한다.
3. 폼을 닫는다.
4. 저장소 데이터는 변경하지 않는다.

적용 대상:

- 커스텀 숙제 추가·수정 폼의 `취소`
- 일정 기념일 추가 폼의 `취소`와 헤더 `닫기`
- Home 기념일 추가 폼의 헤더 `닫기`
- 기존 `DdayCard`의 헤더 `닫기`

캐릭터 폼, 이미지 편집기, 주간 메모는 이미 닫기와 초안 폐기가 정상적으로 연결되어 있어 수정하지 않는다. 확인 대화상자의 취소는 입력 폼이 아니므로 범위에서 제외한다.

## 구현 순서

1. `CustomTaskList` 괄호 누락을 수정해 개발 서버 컴파일을 복구한다.
2. 공통 날짜 검증 함수와 단위 테스트를 추가한다.
3. `AnniversaryDateField`를 세 기념일 입력 화면에 연결한다.
4. 커스텀 숙제와 기념일 폼의 dismiss 핸들러를 명시적으로 분리한다.
5. 브라우저에서 원래 재현 절차를 반복한다.

## 회귀 검증

- `9999-12-31`은 허용되고 다섯 자리 이상의 연도, 세 자리 연도, 존재하지 않는 날짜는 거부되는지 확인한다.
- 기념일 저장 후 기존 `YYYY-MM-DD` 데이터와 D-day 계산이 유지되는지 확인한다.
- 커스텀 숙제 추가 및 수정 중 취소하면 폼이 닫히고 저장소가 바뀌지 않는지 확인한다.
- 기념일 폼을 닫고 다시 열면 이전 미저장 초안이 남지 않는지 확인한다.
- 캐릭터 편집 및 주간 메모의 기존 취소 동작에 회귀가 없는지 확인한다.
- 전체 test와 build를 실행한다.
- `/tasks/manage`, `/calendar`, Home을 브라우저에서 확인하고 콘솔 오류가 없는지 점검한다.

## 제외 범위

- 기념일 데이터 형식 또는 마이그레이션 변경
- 날짜 입력 UI를 연·월·일 세 칸으로 분리하는 작업
- 기념일 반복, 알림, 수정 기능 추가
- 확인 대화상자와 저장 즉시 반영형 컨트롤의 동작 변경
