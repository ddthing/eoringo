# 숙제 탭 캐릭터 정보 중복 제거 설계

## 목표

숙제 탭에서 현재 캐릭터의 이름, 서버, 프로필이 반복 노출되는 문제를 줄인다. 사용자는 어느 캐릭터의 숙제를 보고 있는지 즉시 알 수 있어야 하며, 캐릭터 전환 기능과 기존 숙제 기능은 그대로 유지한다.

## 현재 문제

현재 캐릭터 정보가 다음 세 곳에 반복된다.

1. `TaskOverview`의 현재 캐릭터 카드
2. `CharacterSwitcher` 상단의 현재 캐릭터 요약
3. `CharacterSwitcher` 안의 선택된 캐릭터 버튼

작은 모바일 화면에서 진행도보다 캐릭터 정보가 더 큰 시각적 비중을 차지한다.

## 승인된 설계

### 진행도

- `TaskOverview`에서 현재 캐릭터 카드를 제거한다.
- 일일 진행도와 주간 진행도를 2열로 확장한다.
- 기존 퍼센트와 완료 수 표시는 유지한다.

### 캐릭터 전환

- 숙제 탭의 `CharacterSwitcher`에서는 상단 현재 캐릭터 요약을 표시하지 않는다.
- 영역을 `숙제 대상`이라는 짧은 라벨로 식별한다.
- 캐릭터 버튼에는 기존 프로필 이미지와 이름을 유지한다.
- 선택된 버튼은 색상, 테두리, `aria-pressed` 상태로 현재 캐릭터임을 나타낸다.
- 대표 캐릭터 배지는 기존 동작을 유지한다.
- 캐릭터 목록의 가로 스크롤과 전환 기능을 유지한다.

### 정보 원칙

- 현재 캐릭터 이름은 선택된 캐릭터 버튼에서 한 번만 표시한다.
- 서버명은 숙제 탭의 전환 영역에서 제거한다. 서버는 숙제 결과에 영향을 주지 않으며 설정에서 확인할 수 있다.
- 빈 캐릭터 상태와 캐릭터 추가/관리 동작은 기존 컴포넌트 동작을 유지한다.

## 컴포넌트 변경 범위

- `TaskOverview`: 3열에서 2열 진행도 요약으로 변경
- `TaskManagerPage`: 숙제 대상 라벨을 추가하고 `CharacterSwitcher`에 현재 요약 비표시 옵션 전달
- `CharacterSwitcher`: 기존 `showCurrentSummary` 입력을 재사용하며 선택 상태의 시각적 대비만 필요한 범위에서 보강

새 전역 상태, 데이터 마이그레이션, 라우팅 변경은 없다.

## 접근성

- 선택된 캐릭터 버튼의 기존 `aria-pressed`를 유지한다.
- 라벨과 선택 상태를 색상만으로 구분하지 않는다.
- 터치 영역과 키보드 버튼 동작을 유지한다.
- 가로 스크롤 영역은 기존 방식대로 동작한다.

## 검증

- 전체 단위 테스트 실행
- 프로덕션 빌드 실행
- 숙제 탭에서 현재 캐릭터가 한 번만 보이는지 확인
- 캐릭터 전환 후 진행도와 숙제 목록이 선택 캐릭터 기준으로 갱신되는지 확인
- 모바일 폭에서 2열 진행도와 캐릭터 버튼이 넘치지 않는지 확인

## 제외 범위

- 숙제 데이터 구조 변경
- 캐릭터 관리 기능 변경
- 리셋 시간 설계 및 구현
- 숙제 분류 또는 목록 구조 개편
